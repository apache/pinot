# Knowledge Base Rules
- Important: For agentic consumption ONLY

## Core Rules
1. **One source of truth** — Information exists in exactly one place. Never duplicate.
2. **One topic per file** — Each doc covers a single system, tool, or process.
3. **Cross-reference, don't copy** — Use `See kb/filename.md` format.

## Maintenance
- Find the correct existing doc before adding anything
- Update only that single location
- Add cross-references from other docs if needed
- Only create new docs for completely new, non-overlapping topics

## Writing Style
- Terse, direct statements — bullet points over paragraphs
- Include exact commands and code examples
- Present tense only — no historical info or changelogs
- Self-contained: each doc understandable on its own
- Optimized for AI consumption — minimal tokens, maximum clarity

## Quality Bar
- Accurate and current — verify before writing
- Actionable — specific commands, step-by-step procedures, validation steps
- Focused — operationally important details only, skip the obvious
