# Apache Pinot — Claude Development Guide

Practical build and test strategy for iterative development in this repo.

## Targeted rebuild (fastest for iterative dev)

Only rebuild the modules you changed. This skips ~150 unrelated modules.

```bash
./mvnw install -T 1C \
  -pl pinot-spi,pinot-common,pinot-segment-spi,pinot-segment-local,pinot-core,pinot-tools \
  -DskipTests -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true -q
```

Adjust `-pl` to include only the modules you actually touched. Common module
groups:
- SPI / shared: `pinot-spi,pinot-common,pinot-segment-spi`
- Segment / index work: `pinot-segment-local`
- Query execution: `pinot-core`
- Quickstart / CLI: `pinot-tools`

Add `--offline` / `-o` when you have not added new dependencies.

## Running a single unit test

```bash
./mvnw test -pl <module> -Dtest=<TestClassName> \
  -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true
```

Example:
```bash
./mvnw test -pl pinot-segment-local \
  -Dtest=SparseMapSegmentCreationTest \
  -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true
```

Always rebuild the module before running its tests:
```bash
./mvnw install -pl <module> -DskipTests ... && \
./mvnw test   -pl <module> -Dtest=<TestClass> ...
```

## Running a single integration test

Integration tests start a full embedded Pinot cluster (ZK + controller +
broker + server). They are slow; run them only for cross-role validation.

```bash
./mvnw test -pl pinot-integration-tests \
  -Dtest=<IntegrationTestClass> \
  -Dsurefire.failIfNoSpecifiedTests=false \
  -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true
```

## Running the quickstart

Kill any lingering Pinot processes and wipe temp state first, then launch:

```bash
pkill -9 -f "pinot" 2>/dev/null; sleep 2
rm -rf /tmp/QuickStart* /tmp/PinotAdmin /tmp/pinotController /tmp/pinotServer /tmp/pinotBroker /tmp/pinotMinion
build/bin/pinot-admin.sh QuickStart -type <TYPE>
```

Common quickstart types: `BATCH`, `MULTI_STAGE`, `STREAM`, `HYBRID`.

The cluster is ready when the log prints `play around in the query console`.

### Full build for quickstart

```bash
./mvnw clean install -DskipTests -Pbin-dist -Pbuild-shaded-jar -Dskip.npm -q
```

Notes:
- **`-Dskip.npm`** is required on devpods — the `pinot-controller` frontend
  build (`npm ci`) fails due to missing Node.js deps. This is unrelated to
  backend code and safe to skip.
- If the build fails at `pinot-controller` with `npm ci` error, resume with:
  ```bash
  ./mvnw install -DskipTests -Pbin-dist -Pbuild-shaded-jar -Dskip.npm -rf :pinot-controller -q
  ```
- The built distribution lands in `build/` (symlinked from
  `pinot-tools/target/pinot-tools-pkg/`).

### Targeted rebuild for quickstart (faster after initial build)

When iterating on specific modules, rebuild only what changed then re-package:

```bash
# Step 1: rebuild changed modules
./mvnw install -T 1C \
  -pl pinot-spi,pinot-segment-local,pinot-core,pinot-tools \
  -DskipTests -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true -q

# Step 2: rebuild distribution
./mvnw package -pl pinot-tools -Pbin-dist \
  -DskipTests -Dlicense.skip -Dcheckstyle.skip -Drat.ignoreErrors=true -q

# Step 3: copy freshly-built JARs (assembly may use stale copies)
cp pinot-core/target/pinot-core-*.jar          build/lib/
cp pinot-segment-local/target/pinot-segment-local-*.jar build/lib/
cp pinot-spi/target/pinot-spi-*.jar            build/lib/
```

### Testing schema upload against running quickstart

```bash
# Upload schema
curl -X POST "http://localhost:9000/schemas" \
  -H "Content-Type: application/json" -d @schema.json

# Validate schema round-trip (shows what controller would store)
curl -X POST "http://localhost:9000/schemas/validate" \
  -H "Content-Type: application/json" -d @schema.json

# Fetch schema back
curl "http://localhost:9000/schemas/<schemaName>"
```

## Code style

After making Java changes, format before committing:
```bash
./mvnw spotless:apply -pl <module>
./mvnw checkstyle:check -pl <module>
./mvnw license:format   -pl <module>   # only needed for new files
```

Or run all three at once across changed modules:
```bash
./mvnw spotless:apply checkstyle:check \
  -pl pinot-spi,pinot-segment-local,pinot-core \
  -Dlicense.skip
```

**IMPORTANT: Always run `mvn checkstyle:check` before pushing.** CI will fail on
checkstyle violations. Common issues:
- Member fields must start with `_` (pattern: `^_[a-z][a-zA-Z0-9]*$`)
- Import ordering
- Line length (max 120 chars)

## Git remotes and pushing

This repo has multiple remotes:

- **`origin`** — Uber internal (`code.uber.internal:data/pinot`). Push here for internal Phabricator diffs and internal branches.
- **`upstream`** — Apache Pinot (`github.com/apache/pinot`). Read-only; never push directly.
- **`upstream-fork`** — Personal GitHub fork (`github.com/tarun11Mavani/pinot`). Push here for open-source PRs against `apache/pinot`.

**When to use which:**
- Working on an **open-source PR** (e.g. `sparse-map-1-storage`): push to `upstream-fork`
- Working on an **internal diff** or Uber-specific branch: push to `origin`
- Never push to `upstream` directly — submit PRs via `upstream-fork`
